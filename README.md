# Microsoft 365 User Lifecycle & Access Management Lab

## 📌 Overview

This project simulates a real-world IT support environment using Microsoft 365 Admin Center and Microsoft Entra ID. It demonstrates user lifecycle management, security enforcement, and access control following enterprise best practices.

---

## 🎯 Objectives

* Perform user onboarding and offboarding
* Implement Role-Based Access Control (RBAC)
* Enforce Multi-Factor Authentication (MFA)
* Configure enterprise application access (Slack)
* Apply least privilege security principles

---

## 🛠️ Technologies Used

* Microsoft 365 Admin Center
* Microsoft Entra ID (Azure AD)
* Microsoft Teams / SharePoint (conceptual)
* Slack (Enterprise Application)

---

## 🔧 Tasks Performed

### 👤 User Onboarding

* Created users in Microsoft 365
* Assigned licenses
* Configured initial passwords

### 🔐 Security Configuration

* Enabled Multi-Factor Authentication (MFA)
* Applied security best practices

### 🧩 Access Management (RBAC)

* Created Finance Department group
* Assigned users to groups
* Managed permissions using least privilege

### 🔗 Application Access Control

* Integrated Slack as an enterprise application
* Assigned users/groups to application
* Enforced "user assignment required"

### 📦 Bulk User Management

* Created multiple users simultaneously for onboarding simulation

### ❌ User Offboarding

* Blocked sign-in access
* Removed user access to resources
* Secured accounts to prevent unauthorized access
